# NextGen Systems – Information Systems Project

## Overview

This repository contains an academic project for the course **Information Systems**.

The project is based on the creation and presentation of a fictional IT consulting company named **NextGen Systems**. The work focuses on the use of different information system tools and platforms, including project management tools, website development, business process simulation, CRM-related tasks, data visualization, and data mining.

## Company Profile

**Company Name:** NextGen Systems
**Slogan:** Η τεχνολογία του Αύριο, Σήμερα!
**Business Sector:** IT Consulting Systems

NextGen Systems is a fictional IT consulting company that provides technology-based solutions to businesses. Its goal is to support companies in their digital transformation journey by offering modern IT services, cloud solutions, cybersecurity support, business intelligence tools, and infrastructure management.

## Project Goals

The main goal of this project was to become familiar with different information systems and digital business tools.

More specifically, the project involved:

* Organizing teamwork using project management platforms
* Creating a company profile and brand identity
* Designing a WordPress website
* Simulating business processes
* Completing Salesforce Trailhead tasks
* Creating Tableau dashboards
* Building a machine learning model using RapidMiner
* Practicing collaboration, planning, and time management

## Tools & Platforms Used

The project made use of the following tools and platforms:

* **Confluence** – Team documentation and project coordination
* **Jira** – Task management and Kanban board organization
* **WordPress** – Website creation for the fictional company
* **Signavio** – Business process modeling and simulation
* **Salesforce Trailhead** – CRM-related learning tasks
* **Tableau Public** – Data visualization and dashboards
* **RapidMiner** – Data mining and decision tree classification

### 4. Business Process Simulation with Signavio

The project included a business process simulation using Signavio.

Two different scenarios were compared:

* Scenario with 50 daily requests
* Scenario with 100 daily requests

The analysis showed that increasing the number of daily requests caused a disproportionate increase in waiting times. The simulation helped demonstrate how resource limitations and process bottlenecks can affect business performance.

### 6. Tableau Dashboards

Tableau was used to create data visualizations for two different datasets.

#### Wine Dataset

The wine dataset analysis included visualizations about:

* Relationship between wine price and rating
* Average wine price by country
* Average wine rating by country

The goal was to identify trends between price, quality, and country of origin.

#### IMDb Dataset

The IMDb dataset analysis included visualizations about:

* Relationship between movie duration and IMDb score
* Movie genres and total revenue
* Relationship between movie age and IMDb score

The goal was to understand how different movie characteristics relate to ratings and commercial success.

### 7. RapidMiner Data Mining

RapidMiner was used to build and evaluate a decision tree classification model.

The dataset used was the **mushrooms.csv** dataset from Kaggle. The goal was to classify mushrooms as edible or poisonous based on their characteristics.

The model used:

* Select Attributes
* Set Role
* Cross Validation
* Decision Tree
* Apply Model
* Performance Evaluation
* Confusion Matrix

The first experiment achieved high accuracy, showing that the model could classify mushrooms effectively. The most important feature for classification was found to be **odor**, followed by other characteristics such as spore print color, stalk color, stalk root, and habitat.
